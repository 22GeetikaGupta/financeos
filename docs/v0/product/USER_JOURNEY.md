# 1. Introduction

This document describes how users interact with FinanceOS from onboarding to making financial decisions.

Unlike the System HLD, which focuses on system architecture, this document focuses on the end-user experience.

It defines:

* Who the user is
* What information the user provides
* What the application stores
* How simulations are performed
* How users compare financial strategies
* How recommendations are generated

---

# 2. Target User

FinanceOS is designed for individuals who want to make informed financial decisions based on their complete financial profile.

Typical users include:

* Salaried professionals
* Home loan borrowers
* Individuals planning long-term investments
* Users managing multiple financial goals

Examples of common questions users want answered:

* Should I prepay my home loan?
* Should I invest in FD or RD?
* Can I afford a new car?
* How much should I save every month?
* Will I reach my retirement goal?
* Which financial strategy gives me the highest net worth?

# 4. User Journey

The typical user journey consists of the following steps.

## Step 1 - User Registration

The user creates an account.

---

## Step 2 - Create Financial Profile

The user provides their financial information.

Examples include:

Personal Details

* Name
* Age
* Retirement Goal

Income

* Salary
* Expected Salary Growth
* Bonus (Optional)

Expenses

* Monthly Expenses
* Existing EMIs

Assets

* Bank Balance
* Fixed Deposits
* Recurring Deposits

Liabilities

* Home Loan
* Car Loan
* Personal Loan

Goals

* Buy House
* Buy Car
* Retirement
* Emergency Fund

The financial profile is stored and becomes the user's baseline profile.

---

## Step 3 - Dashboard

After completing the profile, the user lands on the dashboard.

The dashboard provides a consolidated view of the user's financial position.

Examples include:

* Net Worth
* Outstanding Loans
* Monthly Cash Flow
* Investment Summary
* Financial Goals
* Recent Simulations

---

## Step 4 - Run Simulation

The user creates a new simulation.

A simulation starts with the user's existing Financial Profile.

The user then modifies one or more values without changing the original profile.

Examples:

* Increase FD contribution
* Add a loan prepayment
* Increase salary growth
* Increase monthly expenses

These modifications form a simulation scenario.

---

## Step 5 - View Simulation Results

After execution, FinanceOS presents:

* Loan Progress
* Investment Growth
* Cash Flow
* Net Worth Timeline
* Goal Progress
* Summary Metrics

The user can analyze how the selected strategy performs over time.

---

## Step 6 - Save Simulation

Users can save simulation results.

Saved simulations remain available for future comparison.

---

## Step 7 - Compare Strategies

Users can compare multiple saved simulations.

Example comparison metrics:

* Final Net Worth
* Total Interest Paid
* Total Investment Corpus
* Loan Closure Date
* Cash Balance
* Goal Achievement

This allows users to identify the strategy that best meets their financial objectives.

---

## Step 8 - Receive Recommendations

FinanceOS analyzes the user's profile and simulations to generate personalized recommendations.

Examples:

* Increase loan prepayments.
* Increase emergency fund.
* Delay discretionary spending.
* Increase monthly investments.
* Improve savings rate.

Recommendations are based on the user's financial profile and simulation outcomes.

---

# 5. Financial Profile

The Financial Profile is the permanent representation of the user's financial information.

It acts as the baseline for every simulation.

A Financial Profile consists of:

Personal Information

Income

Expenses

Assets

Liabilities

Goals

Financial Assumptions

Updating the Financial Profile affects future simulations but does not modify previously saved simulations.

# 8. Simulation Workflow

Every simulation follows the same workflow.

Financial Profile

↓

Scenario Modifications

↓

Simulation Engine

↓

Simulation Results

↓

Save Simulation

↓

Compare with Previous Simulations

This workflow ensures that the user's original financial profile remains unchanged while allowing unlimited "what-if" analyses.

---

# 9. Recommendation Workflow

Recommendations are generated after simulations complete.

The recommendation engine evaluates:

* Financial Profile
* Simulation Results
* Financial Goals

It then generates personalized suggestions to help users improve their long-term financial outcomes.

Version 1 recommendations are based on simulation data only.

Future versions may incorporate external financial data and market conditions.

# Summary

FinanceOS is designed around a persistent Financial Profile that serves as the foundation for all financial simulations.

Users create and maintain their financial profile once, explore multiple financial scenarios through simulations, compare strategies over time, and receive personalized recommendations based on their financial goals.

This approach transforms FinanceOS from a collection of financial calculators into a comprehensive financial planning platform centered on informed decision-making.
